# Shopaware Pulse Overage Explorer Requirements

## Functional Requirements

- Expose a small HTTP API that the frontend can call.
- Implement multi-tenant isolation via request header: X-Tenant-Id .
- Requests without it must fail.
- Cross-tenant access must not be possible.
- Load seed data from a local JSON file (in-memory is fine).
- Calculate usage totals + overage charges for the billing cycle.
- Provide a way to create and list internal support notes per shop.
- Provide an OpenAPI spec (openapi.yaml/openapi.json) that matches what you implemented.
- Provide a simple health endpoint.

## Non-Functional Requirements

## API Contracts

### GET /api/shops

#### Request

- Method: GET
- URL: /api/shops
- Query Parameters: search, plan, status and sort
- Headers: X-Tenant-Id (required)

#### Response

- Status Code: 200 OK
- Body: JSON array of shops with the following fields:
  - id: string
  - name: string
  - plan: string
  - status: string
  - usage: number
  - included_usage: number
  - overage_charges: number
  - created_at: string (ISO 8601 format)
  - updated_at: string (ISO 8601 format)

### GET /api/shops/{shopId}

#### Request

- Method: GET
- URL: /api/shops/{shopId}
- Headers: X-Tenant-Id (required)

#### Response

- Status Code: 200 OK
- Body: JSON object with the following fields:
  - id: string
  - usage: []UsageEntry
  - notes: []SupportNote
    - id: string
    - author: object
      - id: string (pending to confirm)
      - name: string (pending to confirm)
    - content: string
  - billing_cycle_start: string (ISO 8601 format)
  - billing_cycle_end: string (ISO 8601 format)
  - created_at: string (ISO 8601 format)
  - updated_at: string (ISO 8601 format)

### POST /api/shops/{shopId}/notes

#### Request

- Method: POST
- URL: /api/shops/{shopId}/notes
- Headers: X-Tenant-Id (required)
- Body: JSON object with the following fields:
  - author: object
    - id: string (pending to confirm)
    - name: string (pending to confirm)
  - content: string

#### Response

- Status Code: 204 No Content

### GET /health

#### Request

- Method: GET
- URL: /health
- Headers: None
- Query Parameters: None
- Body: None

#### Response

- Status Code: 200 OK
- Body: JSON object with the following fields:
  - status: string (e.g. "ok")
  - timestamp: string (ISO 8601 format)
  - version: string (application version, e.g. "1.0.0")
- Status Code 503 Service Unavailable (if the application is not healthy)
- Body: JSON object with the following fields:
  - status: string (e.g. "unhealthy")
  - timestamp: string (ISO 8601 format)
  - version: string (application version, e.g. "1.0.0")
- Status Code 500 Internal Server Error (if there is an unexpected error)
- Body: JSON object with the following fields:
  - status: string (e.g. "error")
  - timestamp: string (ISO 8601 format)
  - version: string (application version, e.g. "1.0.0")
